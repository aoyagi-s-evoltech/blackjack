## 💻 プロジェクト概要
[プロジェクト概要を記載]
C#の学習目的として作成した、コンソール上でディーラーと対戦できるブラックジャックです。
基本的な「Hit（カードを引く）」「Stand（勝負する）」の機能に加え、
エースの数値を状況に応じて「1」または「11」として扱うロジックなどを実装しています。
ブラックジャックゲームを作成。
特に「エースの扱い」や「入力チェック」などを実装することで、単なる学習用コードではなく、実際に遊べるゲームとして仕上げています。

## 📐 システム設計

[クラス設計など、設計について説明を記載]
GameManager: ゲーム全体の進行を管理（StartGame / PlayerTurn / DealerTurn / JudgeWinner）
Deck: デッキを管理（カード生成・シャッフル・DrawCard）
Card: カードの情報（数字・カードのランク）
Player/Dealer: プレイヤーとディーラーの手札・行動を管理
PlayerBase: 共通の動作を行うクラスとして手札操作を定義

設計では「単一責任の原則」を意識し、クラスごとに役割を明確に分けました。
GameManager はゲーム全体の進行を管理し、プレイヤーやディーラーの行動を呼び出す役割に限定しています。
Player と Dealer はそれぞれの行動を持ちますが、共通部分は PlayerBase にまとめ、継承を用いて重複を減らしました。
これにより、コードの再利用性が高まり、将来的に別のゲームを追加する際にも柔軟に対応できます。

## 💡 工夫した点・技術的アピール

[開発において意識した点、工夫した点などアピールポイントを記載]
1. 責務分離と拡張性
• GameManager にゲーム進行を集中させ、プレイヤーやディーラーの行動はそれぞれのクラスに分けて処理。
• これにより「新しいルール追加」や「別ゲームへの応用」が容易になる設計を意識しました。

2. 入力チェックとユーザー体験
• ユーザーが誤ったキーを入力してもゲームが落ちないように例外処理を導入。
• 再入力を促すことで「遊びやすさ」を確保しました。

3. 命名規則と可読性
• メソッド名は「動詞＋名詞」（例: DrawCard）で統一。
• StringBuilder を活用し、複数行の表示をまとめて処理

4. ドキュメントコメントとレビュー性
• ドキュメントコメントを活用し、メソッドの役割や引数の意味を明示。コードの意図やメソッドの役割が一目で分かるように工夫しました。
• 他者レビューを意識して「コードの意図が一目で分かる」状態を目指しました。

5. 動く形としてのアウトプット
• 実際に遊べるゲームを作ることで、学習成果を「動く形」に残すことを意識しました。
• オブジェクト指向設計の理解を確認できるだけでなく、他者に共有して遊んでもらえる「体験」としてアウトプットできています。
