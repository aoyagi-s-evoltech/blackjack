## 💻 プロジェクト概要
このプロジェクトは、C# の学習を目的として作成した
コンソール版のブラックジャックゲームです。

プレイヤーとディーラーがカードを引き合い、
21 に近い方が勝ちになるという、ブラックジャックの基本的な流れを
コンソール上で体験できるように作っています。

Hit（カードを引く）や Stand（勝負する）といった基本操作に加えて、
デッキの生成・シャッフル、バースト判定、勝敗判定など、
ゲームとして必要な処理を一通り実装しています。

また、ユーザーが誤った入力をしてもゲームが止まらないように、
入力チェックや再入力の仕組みを丁寧に組み込み、
「実際に遊べる」ことを大切にしています。

コードの構造にも気を配り、
GameManager・Player・Dealer・Deck などのクラスを分けて
役割をはっきりさせることで、読みやすさと保守性を意識した作りにしています。

## 📐 システム設計

このブラックジャックは、クラスごとに役割を分けて作っています。
「このクラスは何をするのか」が分かりやすいように意識して設計しました。
それぞれが担当する仕事をはっきりさせることで、コードが読みやすくなり、
あとから機能を追加したり、修正したりしやすい形になっています。

### 🔸 クラスの役割

- **GameManager**  
  ゲーム全体の流れをまとめるクラスです。  
  最初の手札の配布、プレイヤーやディーラーのターンを進行、  
  勝敗判定を担当します。

- **Deck**  
  デッキ（カードの束）を作成、シャッフル、  
  1枚ずつカードを引く処理を担当します。

- **Card**  
  カード1枚を表すクラスです。  
  数字やランクなど、カードに必要な情報だけを持っています。

- **PlayerBase**  
  プレイヤーとディーラーの共通部分をまとめたクラスです。  
  手札を持つことや、点数を計算する処理などを担当します。

- **Player / Dealer**  
  PlayerBaseを継承し、共通の手札管理や点数計算の仕組みを受け継いだクラスです。

  **Player** はユーザーの入力に応じて行動するクラスで、  
  Hit（カードを引く）か Stand（勝負する）かを選べるようになっています。

  **Dealer** はブラックジャックのルールに従い、  
  17点以上になるまで自動でカードを引く動きを持っています。

  どちらもPlayerBaseの機能を使いながら、  
  それぞれの役割に必要な処理だけを追加する形で実装しています。

### 🔸 設計で大切にしたこと

- **役割をひとつにしぼること**  
  1つのクラスが複数の責務を持たないようにして、  
  「このクラスは何をするのか」が分かりやすいようにしました。

- **表示とロジックを分けること**  
  画面に出す処理はGameManagerにまとめ、  
  プレイヤーやディーラーは"ゲームの中の動き"だけを担当します。  
  こうすることで、あとから画面を変えたいときも楽になります。

- **共通部分はまとめること**  
  PlayerとDealerの似ている部分はPlayerBaseにまとめて、  
  同じコードを書かなくていいようにしました。

- **安全に使えるようにすること**  
  Deck/Player/Dealerはゲーム中に変わらないのでreadonlyにして、  
  誤って書き換えられないようにしています。

- **手札の公開範囲を適切に制御すること**  
  Handをprivateにし、  
  表示用メソッド（GetHandString/GetFirstCardRank）で必要な情報だけ取得できるようにしました。

全体として、「読みやすく、あとから修正しやすいコード」を目指して作成しています。

## 💡 工夫した点・技術的アピール

このプロジェクトでは、「読みやすさ」と「安心して遊べること」を大切にしながら、
コードの書き方や設計にも気を配って作りました。

### 1. 責務を分けて、分かりやすい構造にしたこと
GameManager はゲーム全体の流れだけを担当し、  
プレイヤーやディーラーの動きはそれぞれのクラスに分けて実装しました。  
役割をはっきりさせることで、あとからルールを追加したり、  
別のゲームに応用したりしやすい形になっています。  
オブジェクト指向の基本である「責務の分離」を意識して設計しました。

### 2. 入力チェックで「遊びやすさ」を大切にしたこと
ユーザーが間違った入力をしてもゲームが止まらないように、  
入力チェックや再入力の仕組みを入れています。  
Hit/Standはyes/noのみ受け付け、  
再プレイの確認は「yes以外は終了」という仕様にしています。  
コンソールゲームでもストレスなく遊べるように工夫しました。

### 3. 名前の付け方や書き方をそろえて、読みやすくしたこと
メソッド名は「動詞＋名詞」（例：DrawCard）の形で統一し、  
何をするメソッドなのかが分かりやすいようにしました。  
また、StringBuilder を使って複数行の表示をまとめるなど、  
コードの見通しが良くなるように意識しています。

### 4. ドキュメントコメントで意図を伝えやすくしたこと
各メソッドにドキュメントコメントを付けて、  
「このメソッドは何をするのか」「引数は何を意味するのか」が  
読み手に伝わるようにしました。  
他の人が読んだときにも理解しやすいコードを目指しています。

### 5. 実際に遊べる形でアウトプットしたこと
学んだ内容を「動く形」で残すことを大切にし、  
実際に遊べるブラックジャックとして完成させました。  
オブジェクト指向の理解を深めるだけでなく、  
他の人にも遊んでもらえる作品として仕上げています。

## 🔮 今後の展望

このブラックジャックは、まず「遊べる形」を目指して作りましたが、  
今後はさらに機能を増やしたり、見た目を分かりやすくしたりして、  
もっと楽しめるゲームにしていきたいと考えています。

- **エースの扱いをより本格的にすること**  
  エースを状況に応じて「1」と「11」で切り替えるロジックを追加し、  
  よりブラックジャックらしい計算ができるようにしたいです。

- **UI を強化すること**  
  コンソールだけでなく、WPF や Windows Forms を使って  
  カード画像を表示したり、視覚的に分かりやすい画面にしたいです。

- **チップや賭け金のシステムを追加すること**  
  ベット機能を入れて、ゲーム性を高めたいと考えています。

- **複数プレイヤーへの対応**  
  PlayerBase の仕組みを活かして、  
  プレイヤーを増やせるようにすることも視野に入れています。

学習を続けながら、少しずつ機能を追加して、  
より楽しく、より分かりやすいゲームに改良していく予定です。
